Background Info

  • Uses SQLite in the browser, so it doesn't work exactly like normal databases, but its close.
  • Shows the query and results in real time, demonstrating the parsing of the query on the backend as well as what results are returned from the database.
  • Error messages mainly come from SQL-js, which are not particularly helpful for creating Error-based SQL Injections.
  • SQLite doesn't compare integers and strings, so the classic 
    ' or 1='1
    has to be something like 
    ' or 1=0+'1
